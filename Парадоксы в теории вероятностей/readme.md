# Парадокс Монти Холла

## Суть парадокса

Парадокс Монти Холла - это знаменитая вероятностная задача, названная в честь ведущего одноименного игрового шоу.

### Условия игры:
1. Есть 3 двери: за одной - автомобиль, за двумя другими - козлы
2. Игрок выбирает одну дверь
3. Ведущий, который знает, где приз, открывает одну из оставшихся дверей с козлом
4. Игроку предлагается выбор: остаться при своем выборе или поменять дверь

### Вопрос:
Что выгоднее - менять выбор или нет?

## Интуитивное заблуждение

Большинство людей считают, что:
- После открытия одной двери с козлом шансы становятся 50/50
- Нет разницы, менять выбор или нет

## Математический подход

**Теоретические вероятности:**

- Стратегия "НЕ менять": вероятность выигрыша = 1/3 ≈ 0.333
- Стратегия "МЕНЯТЬ": вероятность выигрыша = 2/3 ≈ 0.667

**Объяснение:**

- Первоначально вероятность выбрать автомобиль = 1/3
- Если вы выбрали козла (вероятность 2/3), то ведущий всегда откроет другого козла, и смена даст автомобиль
- Таким образом, смена выбора выигрывает в 2 случаях из 3



## Результат
- Без смены: вероятность 1/3 ≈ 0.3333
- Со сменой: вероятность 2/3 ≈ 0.6667

## Как работает программа
1. Программа моделирует указанное количество `num_trials` игр Монти Холла двумя способами, то есть каждая игра проходит с измерением выбора игрока и без него
2. Собирается статистка по выигранным играм при условии изменения выбора и при условии неизменения выбора
3. Полученные данные выводятся в консоль и сравниваются с теоретическими
4. Данные визуализируются с помощью диаграммы 

### Функции:
- `simulate_monty_hall()` - моделирует одну игру
- `run_experiment()` - проводит серию испытаний
- `visualize_results()` - строит графики результатов
